SECURITY REQUIREMENTS:

Password Requirements:

- Minimum 8 characters
- At least 1 number
- At least 1 special character
- At least 1 uppercase letter
- Cannot contain email
- Cannot be common password (check against list)
- Cannot reuse last 5 passwords

Password Hashing:

- Use bcrypt with salt rounds 12+
- Or Argon2id (preferred)
- Never store plain text
- Never log passwords

Token Security:

- JWT with RS256 or HS512
- Access token: 15 minutes
- Refresh token: 30 days
- Rotate refresh tokens on use
- Store refresh token hash in DB
- Single-use password reset tokens
- Reset tokens expire in 1 hour

Rate Limiting:

- Login attempts: 5 per 15 minutes
- Password reset: 3 per hour
- 2FA attempts: 5 per 15 minutes
- API calls: 100 per minute per user

Session Management:

- Track device info and IP
- Show active sessions to user
- Allow user to revoke sessions
- Auto-expire inactive sessions (30 days)
- Invalidate all sessions on password change
- Optional: limit concurrent sessions

Data Protection:

- All data encrypted in transit (TLS 1.3)
- Sensitive data encrypted at rest
- PII redacted in logs
- Tenant isolation enforced
- No SQL injection (use parameterized queries)
- CSRF protection
- XSS prevention

Audit Logging:

- Log all authentication events
- Log permission changes
- Log data access (for sensitive data)
- Log failed attempts
- Include IP, user agent, timestamp
- Retention: 1 year minimum
